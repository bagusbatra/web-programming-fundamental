# 📝 Latihan

## 🎯 Tujuan

- Mengubah layout menjadi reusable component
- Memahami konsep component-based UI
- Menghindari penulisan CSS berulang
- Membuat UI yang konsisten & scalable
- Mulai berpikir seperti frontend developer profesional

---

## 📌 Instruksi

Gunakan hasil dari **Latihan 03 (Responsive Website)**, lalu lakukan refactor menjadi **Reusable UI System**.

---

### 1. Refactor Card menjadi Component

Dari latihan sebelumnya:
- Banyak card dengan style mirip

Sekarang:
- Gunakan **1 class `.card` saja**

Semua card:
- Content
- Project
- Hobbies

👉 Harus menggunakan style yang sama (reusable)

---

### 2. Buat Component Button

Semua tombol:
- Gunakan class `.btn`

Tidak boleh:
- Style button satu per satu ❌

Harus:
- Reusable ✅

Tambahkan:
- Hover effect

---

### 3. Buat Component Section

Semua section:
- Gunakan class `.section`

Tujuan:
- Spacing konsisten
- Layout lebih rapi

---

### 4. Gunakan Utility Class

Buat minimal 2:

Contoh:
- `.text-center`
- `.mt-20`

Gunakan di beberapa tempat (jangan cuma dibuat tapi tidak dipakai)

---

### 5. Konsistensi UI (WAJIB)

Pastikan:
- Padding sama
- Border radius sama
- Warna konsisten
- Jarak antar section rapi

---

### 6. Tetap Responsive

Semua perubahan:
- Tidak boleh merusak responsive

Test:
- HP
- Tablet
- Desktop

---

## ⭐ Challenge

🔥 Level Up (Wajib untuk siswa cepat):

### 1. Variasi Component (Advanced)

Buat variasi:

- `.card-primary`
- `.card-secondary`

Perbedaannya:
- Warna
- Shadow
- Border

---

### 2. Button Variasi

Tambahkan:
- `.btn-primary`
- `.btn-outline`

---

### 3. Tambahkan Hover Animation Lebih Smooth

Gunakan:
- `transform`
- `transition`

Contoh:
- Card naik
- Button membesar sedikit

---

### 4. Buat Layout Lebih “Clean UI”

Perbaiki:
- Spacing
- Alignment
- Hierarchy

👉 Fokus ke tampilan, bukan hanya fungsi

---

## 💡 Tips

- Jangan ulang CSS
- Gunakan class umum
- Pikirkan:
  > “Kalau saya punya 100 card, apakah CSS saya masih rapi?”
